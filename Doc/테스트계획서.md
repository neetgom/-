# 원신 6.3 업데이트 '달 결정' 기능 테스트 계획서

<div align="left">
  
**프로젝트** : 원신 (Genshin Impact)<br>
**작성일** : 2026. 01. 24<br>
**작성자** : 이윤환<br>
**테스트 환경** : Galaxy S25 Ultra (Android 16)<br>

# 1. 테스트 목적
 본 문서는 원신 6.3 버전에 업데이트된 '달 결정(물+바위 반응)' 시스템이 업데이트 공지(패치노트) 내용대로 정확하게 작동하는지 확인하기 위해 작성되었습니다.<br>
신입 QA 지원자로서 유저 입장에서 발생할 수 있는 기능적 오류를 찾아내고, 게임의 규칙이 올바르게 적용되는지 꼼꼼하게 확인하는 것을 목표로 합니다.

# 2. 테스트 범위
제 개인 기기(모바일)로 수행 가능한 기능적인 요소를 중점적으로 테스트합니다.
(※ 기기 발열, 렉(Frame Drop), 네트워크 지연 등 하드웨어 성능 관련 테스트는 제외합니다.)
중점 확인 항목 (Test List)
구분	테스트 항목	확인 내용
1. 발동 조건	물+바위 반응	- 물과 바위 원소가 만났을 때 반응이 즉시 일어나는가?

- 순서(물 먼저 vs 바위 먼저)가 바뀌어도 똑같이 발동하는가?
	방해 요소 확인	- 불, 얼음, 번개 등 다른 원소가 끼어들었을 때 오작동하지 않는가?

- (예: 불 원소가 있으면 '증발'이 먼저 일어나야 함)
2. 조각 생성	달빛 조각	- 반응 성공 시 '달빛 조각'이 필드에 1개 생성되는가?

- 조각이 생성되는 위치가 너무 멀거나 이상한 곳은 아닌가?

- 10~15초 동안 먹지 않으면 자동으로 사라지는가?
3. 조각 수집	획득 및 누적	- 캐릭터가 근처(약 5m)에 가면 자동으로 먹어지는가?

- 현재 나와있는 캐릭터만 먹을 수 있는가? (대기 캐릭터 불가 확인)

- 조각이 최대 3개까지 캐릭터 주위에 쌓이는가?
4. 자동 공격	3스택 발동	- 조각 3개를 모으는 즉시 자동으로 공격이 나가는가?

- 공격이 허공이 아니라 '가장 가까운 적'을 향해 날아가는가?

- 여러 마리의 적이 있을 때 엉뚱한 곳을 때리지 않는가?
5. 데미지	수치 확인	- '원소 마스터리'를 높였을 때 데미지가 올라가는가?

- '치명타'가 터졌을 때 데미지 숫자가 더 크게 뜨는가?

- 적의 속성(바위 내성)에 따라 데미지가 감소하는가?
6. 기타	속성 판정	- 달빛 조각이 '바위 창조물'로 인식되어 다른 캐릭터(치오리 등) 스킬과 연계되는가?

# 3. 테스트 준비물 (도구)

테스트를 체계적으로 관리하고, 실무 프로세스에 맞춘 산출물을 만들기 위해 다음 도구들을 활용합니다.<br>
●	게임 정보: 6.3 업데이트 패치노트 및 이벤트 공지사항 (기능의 기준점)
●	테스트 기기: Galaxy S25 Ultra (최신 안드로이드 환경)
●	기록 및 관리 도구:
○	테스트 케이스(TC) 양식: 테스트 절차(Step)와 기대 결과(Expected Result)를 명확히 구분할 수 있는 구조화된 템플릿(GitHub의 Jira-Zephyr 스타일 양식 활용) 사용
○	GitHub: 작성된 TC와 테스트 결과(Pass/Fail) 파일을 저장소에 업로드하여 버전 관리 및 포트폴리오화
○	캡쳐보드 (Elgato): 모바일 화면을 PC로 송출하여 고화질 녹화 및 프레임 단위 정밀 분석

# 4. 테스트 진행 일정 (계획)

환경 설정이 완료된 상태이므로, 집중적인 테스트 수행을 통해 총 4일 내에 검증을 완료하는 것을 목표로 합니다.<br>
-	사전 준비 (완료): 테스트 계정(물/바위 캐릭터, 성유물 세팅) 및 캡쳐보드 녹화 환경 구축 완료
  -	1일차 (설계): 업데이트 내용을 분석하고, 구조화된 TC 양식에 맞춰 테스트 케이스를 상세히 작성하여 GitHub에 업로드합니다.
  -	2일차 (수행 1 - 기본): 작성한 TC를 기반으로 기본 동작(발동, 수집, 3스택 타격)을 집중 확인하고 결과를 기록합니다.
  -	3일차 (수행 2 - 심화): 데미지 수치 검증 및 방해 요소(타 원소 개입) 등 예외 상황을 테스트하고, 발견된 버그 영상을 분석합니다.
  -	4일차 (보고): 최종 테스트 결과를 정리하고 문서를 마무리합니다.

# 5. 예상되는 어려움 및 주의사항 (Risks)

테스트를 진행하면서 발생할 수 있는 문제점과 해결 방안을 미리 정리했습니다.
예상되는 문제	해결 방안
눈으로 확인하기 어려움	데미지 숫자나 이펙트가 겹쳐서 잘 안 보일 수 있습니다.

→ 캡쳐보드로 PC에 녹화한 뒤, 큰 화면에서 프레임 단위로 돌려보며 확인하겠습니다.
위치가 랜덤함	조각이 떨어지는 위치가 매번 달라서 헷갈릴 수 있습니다.

→ 한두 번만 해보지 않고, 20번 이상 반복해서 공통된 규칙을 찾겠습니다.
모바일 조작 실수	터치 조작 미스로 인해 스킬 쿨타임 등을 잘못 잴 수 있습니다.

→ 조작이 쉬운 '무상의 바위' 같은 고정된 보스 몬스터를 대상으로 테스트하겠습니다.
정보 부족	정확한 지속시간(초)이 공지에 없을 수 있습니다.

→ 녹화된 영상의 타임라인을 확인하여 정확한 시간(ms 단위)을 측정하겠습니다.
